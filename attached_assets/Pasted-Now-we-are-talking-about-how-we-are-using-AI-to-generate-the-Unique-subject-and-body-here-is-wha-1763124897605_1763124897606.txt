How Strategy Messages Currently Work
Here's the exact flow:

1. Storage
The strategyTranscript is stored as JSONB in the sequences table:

{
  messages: [
    { role: 'user', content: 'I want emails that...', createdAt: '...' },
    { role: 'assistant', content: 'Great! Here's my approach...', createdAt: '...' },
    { role: 'user', content: 'Actually, change it to...', createdAt: '...' },
    { role: 'assistant', content: 'Updated strategy...', createdAt: '...' },
    // ... 6, 10, 50 messages...
  ],
  summary: { goals: '...', audience: '...', tone: '...' }, // Optional, not currently used
  lastUpdatedAt: '...'
}

2. Email Generation (The Problem)
Every time an email is generated for ANY recipient, the system:

Takes ALL strategy messages:

if (strategyTranscript?.messages && strategyTranscript.messages.length > 0) {
  strategyContext = '\n\nCAMPAIGN STRATEGY CONTEXT:\n';
  strategyTranscript.messages.forEach(msg => {
    strategyContext += `${msg.role === 'user' ? 'YOU' : 'ASSISTANT'}: ${msg.content}\n`;
  });
}

Injects the entire conversation into the OpenAI system prompt:

${strategyContext}  // ← ALL your back-and-forth messages

3. The Inefficiency
6 messages = ~1,000 tokens injected per email
50 messages = ~8,000+ tokens injected per email
If you send 100 emails, that's 800,000 tokens just replaying your strategy conversation
The AI has to wade through your entire negotiation to find the final strategy
You're Right: We Need a "Finalize Strategy" Feature
Instead of replaying the conversation, you should be able to:

Chat with the AI 3, 6, or 50 times to refine the strategy
Click "Approve Strategy" or "Finalize"
The AI distills the entire conversation into a single, concise campaign brief (200-300 words max)
That brief becomes the finalizedPrompt that gets injected into emails
The conversation history is archived but no longer used
This would:

✅ Cut token usage by 90%+
✅ Make the AI prompts clearer (one directive vs. 50 messages of back-and-forth)
✅ Keep the strategy focused (no dilution from refinement discussions)
✅ Preserve the conversation for audit/review purposes
