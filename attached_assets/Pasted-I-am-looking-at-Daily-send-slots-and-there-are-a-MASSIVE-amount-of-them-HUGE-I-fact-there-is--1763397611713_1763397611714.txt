There's NO duplicate prevention! The table schema from 0022_daily_send_slots.sql doesn't have a unique constraint on (slot_date, slot_time_utc), so it will happily insert duplicate slots.

Root Cause: If ensureDailySlots() gets called multiple times (server restarts, manual triggers, etc.), it will keep inserting 20 new slots each time because:

The duplicate check if (existing.length > 0) only checks if ANY slots exist, not if the right NUMBER exist
