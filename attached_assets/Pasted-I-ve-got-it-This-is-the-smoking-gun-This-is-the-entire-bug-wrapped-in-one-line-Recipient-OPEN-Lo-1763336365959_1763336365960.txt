The correct ISO should be:

That’s not a typo.
That’s formatInTimeZone returning garbage because you’re calling it incorrectly.

The Root Cause (100% certain)

You are using formatInTimeZone() like this:

formatInTimeZone(
  dayLoopDate,
  recipientTimezone,
  `yyyy-MM-ddT${String(openHour)}:${String(openMinute)}:00`
)

That’s invalid.

The pattern string must be a real pattern, not a hybrid pattern with injected hour/minute.

What formatInTimeZone expects:

formatInTimeZone(date, timezone, "yyyy-MM-dd'T'HH:mm:ss")

What you gave it:

"yyyy-MM-ddT11:00:00"

This tells the formatter:

Interpret the “T11:00:00” as a formatting instruction.

Since “11” is not HH tokens and “00” is not mm tokens → the formatter produces garbage.

And then you pass that garbage string to new Date(), creating:

Invalid Date

Once that happens → every later step dies with:

RangeError: Invalid time value

The Fix (exact and minimal)

You must NEVER inject literal hour/minute into the format string.

The correct way is:
	1.	Build the local date by cloning dayLoopDate and setting hours
	2.	Use formatInTimeZone ONLY for conversion into string format

Replace this:

const debugRecipientOpenIso = formatInTimeZone(
  dayLoopDate,
  recipientTimezone,
  `yyyy-MM-ddT${String(openHour).padStart(2,'0')}:${String(openMinute).padStart(2,'0')}:00`
);

With this:

// Clone the day and set hour/min/minute locally
const localOpen = new Date(
  formatInTimeZone(dayLoopDate, recipientTimezone, "yyyy-MM-dd'T'00:00:00")
);
localOpen.setHours(openHour, openMinute, 0, 0);

const debugRecipientOpenIso = formatInTimeZone(
  localOpen,
  recipientTimezone,
  "yyyy-MM-dd'T'HH:mm:ssXXX"
);

And same for closing window.

Why this must be done this way

formatInTimeZone() does NOT generate dates.
It only formats an existing Date object.

You were trying to use it to generate dates.

That will always break.
