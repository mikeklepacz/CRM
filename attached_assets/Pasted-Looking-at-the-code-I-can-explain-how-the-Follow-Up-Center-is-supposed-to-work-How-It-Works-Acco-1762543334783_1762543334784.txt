const claimedUntouched = allClients
  .filter(c => c.claimDate && !c.lastContactDate)

  }));
Reads from:

claimDate - when the store was claimed
lastContactDate - should be null for stores that have never been contacted
Shows: Stores where claimDate exists but lastContactDate is null, calculating days since claim date.

2. Interested Leads Going Cold
const interestedGoingCold = allClients
  .filter(c => {
    const status = (c.data?.Status || c.data?.status || '').toLowerCase();
    return c.lastContactDate && 
           !c.firstOrderDate && 
           (status === 'interested' || status === 'contacted');
  })
  .map(c => ({
    ...c,
    daysSinceContact: Math.floor((now.getTime() - new Date(c.lastContactDate!).getTime()) / (1000 * 60 * 60 * 24))
  }));
Reads from:

lastContactDate - last time contact was made
firstOrderDate - should be null (no order placed yet)
data.Status - should be "interested" or "contacted"
Shows: Stores with a contact date but no order, calculating days since last contact.

3. First-Time Buyers - Reorder Alert
const closedWonReorder = allClients
  .filter(c => c.firstOrderDate && c.firstOrderDate === c.lastOrderDate)
  .map(c => ({
    ...c,
    daysSinceOrder: Math.floor((now.getTime() - new Date(c.firstOrderDate!).getTime()) / (1000 * 60 * 60 * 24))
  }));
Reads from:

firstOrderDate - date of first order
lastOrderDate - date of last order (should equal first order date)
Shows: Stores where first and last order dates are the same (one-time buyers), calculating days since their only order.

Where These Fields Come From
Based on the schema in shared/schema.ts, the clients table has these tracking fields:

claimDate: timestamp("claim_date"),
lastContactDate: timestamp("last_contact_date"), 
firstOrderDate: timestamp("first_order_date"),
lastOrderDate: timestamp("last_order_date"),
These are database fields that should be updated when:

claimDate: Set when an agent claims a store (in the claim/unclaim mutations)
lastContactDate: Should be updated when an agent makes contact (call, email, etc.)
firstOrderDate/lastOrderDate: Updated from WooCommerce order sync
