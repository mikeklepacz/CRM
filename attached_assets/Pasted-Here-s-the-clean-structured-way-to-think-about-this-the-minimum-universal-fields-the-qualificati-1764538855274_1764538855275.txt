III. QUALIFICATION FIELD SET (THE IMPORTANT PART)

You need structured fields that the AI can fill.

1. Eligibility Fields (Tyre Cartel Example)
	•	Bought truck/bus tyres? (yes/no/unsure)
	•	Purchase Years (text or list: 2016–2024)
	•	Brands Purchased (multi-select: Michelin, Goodyear, Continental, Bridgestone…)
	•	Invoices Available? (yes/no/partial)
	•	How many vehicles? (exact or category: 1–10, 10–50, 50–200, 200+)

2. Claim Strength Fields

These are auto-generated by AI:
	•	Eligibility Score (0–100)
	•	Strength Category (A/B/C/D)
	•	Document Status (missing/partial/complete)
	•	Estimated Value Range (optional later)

3. Extra Tenant Fields

Each tenant can add their own custom fields:
	•	“Fleet Manager?”
	•	“Has EU grants?”
	•	“Has prior legal claims?”
	•	etc.

All of these get tied to tenantId.

⸻

IV. QUESTION ENGINE (GENERIC + TENANT SPECIFIC)

You said you’re not sure if you need a “list of questions.”

You do.

But it must be modular, not hard-coded.

Structure:

Questions Table
	•	tenantId
	•	moduleType (sales/qualification)
	•	questionId
	•	questionText
	•	answerType (text, yes/no, number, list, multi-select)
	•	mapsToField (which CRM field this answer populates)
	•	required (yes/no)
	•	order (1,2,3,4…)
	•	agentUsage (AI agent will ask in this sequence)

This lets each tenant define:
	•	Their own question list
	•	Their own field mapping
	•	Their own order
	•	Different call types (sales vs legal intake)

Your AI agent just pulls the question list dynamically.

⸻

V. HOW THE CALL WORKS (AI DRIVEN LOGIC)

1. ElevenLabs makes the call.

Your system already handles:
	•	background audio
	•	transcript
	•	agent persona (Przemek / Holly / etc.)

2. Transcript is saved in the CRM.

Done.

3. OpenAI parses the transcript.

This is the key upgrade.

You will use a generic analysis prompt, which pulls the tenant’s question list and field mappings.

Example:

“Based on this transcript and the tenant’s field definitions, extract the answers to these fields…”

Then OpenAI returns structured JSON:

{
  "boughtTyres": "yes",
  "purchaseYears": "2016-2024",
  "brands": ["Michelin", "Continental"],
  "invoicesAvailable": "partial",
  "fleetSize": "25-100",
  "eligibilityScore": 82,
  "strengthCategory": "A"
}

You save this directly to the Qualification table.

4. CRM auto-generates the quality score.

Simple logic, configurable per tenant:

+20 points for buying tyres
+20 points for years 2016–2024
+20 points for invoices
+20 points for fleet > 10
+10 points for major brands

Let lawyers customize it.
